## 让想法到代码库之间的设计过程不再缺失。

[Design OS](https://buildermethods.com/design-os) 是一个产品规划与设计工具，帮助你定义产品愿景 (Product Vision)、规划数据模型 (Data Shape)、设计 UI，并导出生产就绪的组件用于开发。与其直接开始写代码，不如通过引导式流程捕捉你要构建的内容及其原因——然后将编码代理所需的一切移交给它，以确保正确构建。

## 问题所在

AI 编码工具在快速构建方面表现惊人。但结果往往不尽如人意。你描述需求，代理构建了*一些东西*，但并非你所愿。UI 看起来很平庸。功能只实现了一半。你花在修复和纠偏上的时间，甚至比开发时间还要长。

**核心问题：** 我们要求编码代理在确定“构建什么”的同时完成“构建”。设计决策是临时做出的，埋没在代码中，不推倒重来就无法调整。没有规范，没有共识，也没有关于“完成”标准的唯一事实来源。

## Design OS 流程

Design OS 驱动一个引导式的设计和架构流程。你与 AI 协作，通过结构化的步骤进行工作：

1. **产品规划 (Product Planning)** — 定义愿景，分解路线图，并进行数据模型 (Data Shape) 规划
2. **原子设计与容器设计** — 进行原子设计 (Design Tokens) 与容器设计 (Shell Design)
3. **功能模块设计 (Section Design)** — 针对每个功能区域：指定要求，生成示例数据，并进行界面设计 (Design Screen)
4. **导出 (Export)** — 生成完整的交付包用于实施

每一步都是一场对话。AI 提出问题，你提供指导，共同打造一个符合你愿景的产品——在任何实施开始之前。
